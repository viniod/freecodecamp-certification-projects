Introducao ao Git

Git é um sistema de controle de versão distribuído muito utilizado no desenvolvimento de software. Ele permite acompanhar alterações feitas em arquivos ao longo do tempo. Com ele, desenvolvedores podem trabalhar em conjunto no mesmo projeto sem sobrescrever o trabalho uns dos outros.

Além disso, o Git permite voltar para versões anteriores do código, facilitando a correção de erros ou testes de novas funcionalidades.

Algumas funcionalidades do Git:

  • Controle de versão
  • Histórico de alterações
  • Trabalho em equipe
  • Segurança do código

Aqui está um exemplo da base de um código Git:

git --version
Inicializando um Repositorio

Antes de começar a usar o Git em um projeto, é necessário inicializar um repositório. Isso cria uma pasta oculta onde o Git armazenará todas as informações de controle de versão.

Alguns passos para inicializar um repositório:

  1. Criar um repositório
    2. git init
  2. Monitorar alterações

    3. Após inicializar o repositório, o Git começará a monitorar as alterações feitas nos arquivos do projeto.

    git status
  3. Preparar o projeto para versionamento

Esses são normalmente os primeiros passos quando começamos um novo projeto com Git.

Adicionando e Salvando Alteracoes

Quando um arquivo é modificado, o Git identifica essa mudança. No entanto, para salvar essa alteração no histórico do projeto, é necessário adicioná-la ao estágio de preparação.

  1. Adicionar arquivos ao stage

    2. Primeiro passo é adicionar um arquivo.

    git add arquivo.txt git add
  2. Criar commits

    3. Depois de adicionar os arquivos, utilizamos o comando commit para registrar permanentemente as mudanças.

    git commit -m "Adicionando nova funcionalidade"

    Cada commit representa um ponto específico na história do projeto.

Trabalhando com Branches

Branches são ramificações do projeto principal. Elas permitem desenvolver novas funcionalidades sem afetar a versão principal do código.

Cada branch pode ter alterações diferentes e, quando a funcionalidade estiver pronta, pode ser integrada ao projeto principal. Esse recurso é muito utilizado em equipes de desenvolvimento.

Exemplos:

  • Criar branch

    • git branch nova-feature

  • Alterar branch

    • git checkout nova-feature

  • Mesclar alterações

    • git merge nova-feature

Enviando Codigo para o GitHub

O GitHub é uma plataforma online que hospeda repositórios Git. Ele permite compartilhar código, colaborar com outras pessoas e armazenar projetos na nuvem.

Depois de criar um repositório no GitHub, é possível conectar o projeto local e enviar as alterações para a plataforma. Isso facilita o trabalho em equipe e o backup do código.

Exemplos:

  • Conectar repositório local

    • git remote add origin https://github.com/usuario/repositorio.git

  • Enviar alterações

    • git push origin main

  • Atualizar projeto

    • git pull origin main
